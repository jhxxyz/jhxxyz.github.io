客户端

  2. 连接服务器

  4. 关闭socket

1、建立套接字socket

定义：
建立套接字，返回套接字文件描述符

函数原型：

参数：

  • domain：协议族

    PF_INET / AF_INET -> Ipv4 网络协议
    PF_INET6 / AF_INET6 -> Ipv6 网络协议

  • type：套接字类型

    SOCK_STREAM -> 字节流套接字TCP
    SOCK_DGRAM -> 数据报套接字UDP

  • protocol：协议，0为默认协议

返回值：
成功：套接字文件描述符
失败：-1

连接服务器

函数原型：

参数：

  • sockfd：套接字文件描述符
  • addr：服务器地址和端口号
  • addrlen：地址长度

返回值：
成功：0
失败：-1

示例代码：初始化IP地址和端口号 –IPV4

发送/接收数据

函数原型：

参数：

  • sockfd：套接字文件描述符
  • buf：发送缓冲区
  • len：发送数据长度
  • flags：标志位，默认为0

返回值：

  • 成功：发送的字节
  • 失败：-1

服务器

1、绑定bind
2、监听listen
3、等待连接accept
4、接收数据recv

绑定bind

定义：
绑定自己的IP地址和端口号

函数原型：

参数：

  • sockfd：套接字文件描述符
  • addr：自己的IP地址和端口号
  • addrlen：地址长度

返回值：

  • 成功：0
  • 失败：-1

绑定IP与端口号的结构体（16个字节）： 使用 man 7 ip 可以查看以下结构体

监听listen

定义： 创建一个未连接队列，同时最多连接的客户端总数 backlog。

函数原型：

参数：

  • sockfd：套接字文件描述符
  • backlog：同时最多支持连接上来的客户端总数

返回值：

  • 成功：0
  • 失败：-1

等待连接accept

定义： 等待客户端连接

函数原型：

参数：

  • sockfd：套接字文件描述符
  • addr：客户端的IP地址和端口号
  • addrlen：地址长度

返回值：

  • 成功：已连接的客户端套接字文件描述符
  • 失败：-1

说明：
listen()与accept()是成对出现的，listen()用来创建一个未连接队列，accept()用来接收客户端连接。

接收数据recv

函数原型：

参数：

  • sockfd：套接字文件描述符
  • buf：接收数据存储的缓冲区
  • len：接收数据的大小
  • flags：标志位，一般为0

返回值：

  • 成功：接收到的字节数
  • 失败：-1
  • 0：客户端断开连接

UDP通信——C/S

定义： 相比于TCP,UDP通信的形式更像是发短信。不需要在数据传输之前建立、维护连接，只专心获取数据就好。

客户端

  1. 创建套接字
  2. 绑定IP和端口
  3. 发送数据

一、创建套接字

函数原型：

参数：

  • domain：协议族，

    AF_INET -> IPv4
    AF_INET6 -> IPv6

  • type：套接字类型

    SOCK_STREAM -> TCP
    SOCK_DGRAM -> UDP

  • protocol：一般设为0

返回值：

  • 成功：返回套接字描述符
  • 失败：-1

二、绑定IP和端口

函数原型：

参数：

  • sockfd：套接字描述符
  • addr：自己的IP地址和端口号
  • addrlen：sockaddr结构体的大小

返回值：

  • 成功：0
  • 失败：-1

三、发送数据

函数原型：

参数：

  • sockfd：套接字描述符
  • message：要发送的数据
  • length：要发送的数据的长度（strlen）
  • flags：一般设为0
  • dest_addr：目标IP地址和端口号
  • dest_len：sockaddr结构体的大小

返回值：

  • 成功：发送的字节数
  • 失败：-1

  • 成功：接收到的字节数

特点：

  1. UDP广播通信只能用于局域网，不能用于互联网。
  2. 只有UDP通信才能进行广播通信。

UDP组播通信（群聊）

定义： 组播是介于单播与广播之间，在一个局域网内，将某些主机添加到组中，并设置一个组地址我们只需要将数据发送到组播地址即可，加入到该组的所有主机都能接收到数据。

特点：

    • 需要给组播设置IP地址，该IP必须是D类地址。
  • 只有UDP才能设置组播

IP地址分类：

  • A类地址：1.0.0.1~126.255.255.254
  • B类地址：128.0.0.1~191.255.255.254
  • C类地址：192.0.0.1~223.255.255.254
  • D类地址：224.0.0.1~239.255.255.254
  • E类地址：240.0.0.1~255.255.255.254

#include<stdio.h>
#include <sys/types.h>          /* See NOTES */
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>
#include <string.h>
#include <string.h>
#include <stdlib.h>
#include <unistd.h>

int main(int argc, char *argv[])
{
    //2、定义组播结构体
    struct ip_mreq vmreq;

    if(argc != 4)
    {
        printf("输入格式有误，输入格式:./可执行文件  本机IP  组播IP 端口\n");
        exit(0);
    }    

    //1、创建UDP数据报套接字
    int socketfd = socket(AF_INET,SOCK_DGRAM,0);
    if(socketfd == -1)
    {
        perror("socket error");
        return -1;
    }




    //3、设置组播ip（初始化 组播结构体）
    inet_pton(AF_INET,argv[2],&vmreq.imr_multiaddr); // 组播地址
    inet_pton(AF_INET,argv[1],&vmreq.imr_interface);   // 需要添加到组的ip
    
    //4）加入组播属性（也就是设置这个套接字 可以接收组播信息）
    setsockopt(socketfd,IPPROTO_IP,IP_ADD_MEMBERSHIP,&vmreq,sizeof(vmreq));

    //5）绑定地址
    struct sockaddr_in saddr;
    saddr.sin_family = AF_INET;
    saddr.sin_port = htons(atoi(argv[3]));
    saddr.sin_addr.s_addr = htonl(INADDR_ANY); //htonl(INADDR_ANY)  代表 主机所有的地址

    bind(socketfd,(struct sockaddr *)&saddr,sizeof(saddr));

    //6）接收数据
    struct sockaddr_in otherAddr;
    int len = sizeof(struct sockaddr_in);
    
    while(1)
    {
        char buf[1024]={0};
        
        recvfrom(socketfd,buf,sizeof(buf),0, (struct sockaddr *)&otherAddr,&len);

        printf("来自 %s:%u  recv:%s\n",inet_ntoa(otherAddr.sin_addr),ntohs(otherAddr.sin_port),buf);
    }
    
    //关闭 
    close(socketfd);
    
    return 0;
}
